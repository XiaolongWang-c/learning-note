# 猫狗分类对比实验设计方案

## 1. 任务目标

本任务不是单纯完成猫狗二分类，而是基于同一份数据、同一套训练条件，对比两类网络在训练过程中的表现差异：

- `Plain CNN`：普通卷积堆叠网络
- `Res CNN`：加入残差连接的卷积网络

核心关注点如下：

- 谁收敛更快
- 谁训练更稳定
- 谁在相同 epoch 下取得更高的训练/验证准确率
- 当网络适当加深后，普通 CNN 是否更容易出现优化困难

## 2. 数据集现状

根据当前 `dog-cat` 文件夹的实际内容，数据结构如下：

- 训练目录：`training_set/`
- 测试目录：`test_set/`
- 类别：`cats`、`dogs`

实际统计结果如下：

| 数据划分         | cats | dogs | 说明                                  |
| ------------ | ----:| ----:| ----------------------------------- |
| training_set | 4000 | 2022 | `cats` 目录下额外存在一个 `_DS_Store`，不计入图片数 |
| test_set     | 1011 | 1012 | 基本平衡                                |

结论：

- 当前训练集明显类别不均衡，`cats` 约为 `dogs` 的 1.98 倍。
- 如果直接训练，模型可能偏向预测 `cat`，因此需要在两种模型中统一加入类别不平衡处理策略。

## 3. 实验设计原则

为了保证对比有效，除模型结构外，其余条件全部保持一致：

- 相同输入尺寸
- 相同训练/验证划分方式
- 相同数据增强
- 相同 batch size
- 相同优化器
- 相同学习率与调度策略
- 相同 epoch 数
- 相同随机种子
- 相同类别不平衡处理方式

唯一变化项：

- `Plain CNN` 与 `Res CNN` 的网络结构不同

## 4. 数据划分方案

### 4.1 推荐划分

建议不要把 `test_set` 直接当验证集，而是：

- 从 `training_set` 中按类别分层切分 `80%` 作为训练集、`20%` 作为验证集
- `test_set` 仅在最终模型确定后做一次补充测试

推荐切分结果：

| 子集    | cats | dogs | 合计   |
| ----- | ----:| ----:| ----:|
| Train | 3200 | 1618 | 4818 |
| Val   | 800  | 404  | 1204 |
| Test  | 1011 | 1012 | 2023 |

### 4.2 原因

- 验证集用于观察训练过程中的泛化表现，不能和最终测试混用
- 保留 `test_set` 能让最后结论更完整
- 分层切分可以避免验证集类别分布失真

## 5. 数据预处理与增强

### 5.1 输入规格

- 输入尺寸：`128 x 128`
- 通道数：`3`
- 颜色空间：`RGB`

### 5.2 预处理

- `Resize(128, 128)`
- 转 Tensor
- 标准化：`mean=[0.485, 0.456, 0.406]`，`std=[0.229, 0.224, 0.225]`

### 5.3 数据增强

训练集统一使用：

- `RandomHorizontalFlip(p=0.5)`
- `RandomRotation(10)`
- `ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2)`

验证集和测试集只做：

- `Resize`
- `Normalize`

说明：

- 增强策略不宜过强，否则会干扰“结构差异”这一核心对比变量
- 两个模型必须使用完全一致的数据增强

## 6. 类别不平衡处理

由于训练集 `cat:dog = 4000:2022`，建议在两种模型中统一使用以下方案之一：

### 推荐方案

使用带类别权重的交叉熵损失：

- `weight_cat = 1.0`
- `weight_dog = 4000 / 2022 ≈ 1.98`

### 备选方案

使用 `WeightedRandomSampler`

建议优先采用“加权损失”，原因是：

- 实现简单
- 不改变每个 epoch 的样本总量定义
- 更利于保证两个模型训练过程可直接比较

## 7. 模型设计

### 7.1 Plain CNN

建议设计为 4 个卷积阶段，每阶段 2 个卷积层，不使用跳连。

结构示意：

1. `Conv(3, 32) + BN + ReLU`
2. `Conv(32, 32) + BN + ReLU + MaxPool`
3. `Conv(32, 64) + BN + ReLU`
4. `Conv(64, 64) + BN + ReLU + MaxPool`
5. `Conv(64, 128) + BN + ReLU`
6. `Conv(128, 128) + BN + ReLU + MaxPool`
7. `Conv(128, 256) + BN + ReLU`
8. `Conv(256, 256) + BN + ReLU + MaxPool`
9. `GlobalAveragePooling`
10. `Dropout(0.3)`
11. `Linear(256, 2)`

设计目的：

- 保持网络具有一定深度，足以观察普通堆叠网络的优化特征
- 参数量适中，适合当前数据规模

### 7.2 Res CNN

在与 Plain CNN 接近的通道规模下，改为残差块结构。

结构示意：

1. `Stem: Conv(3, 32) + BN + ReLU`
2. Stage1: `Residual Block(32 -> 32) x 2`
3. Stage2: `Residual Block(32 -> 64, stride=2) + Residual Block(64 -> 64)`
4. Stage3: `Residual Block(64 -> 128, stride=2) + Residual Block(128 -> 128)`
5. Stage4: `Residual Block(128 -> 256, stride=2) + Residual Block(256 -> 256)`
6. `GlobalAveragePooling`
7. `Dropout(0.3)`
8. `Linear(256, 2)`

残差块形式：

- 主分支：`Conv-BN-ReLU-Conv-BN`
- 捷径分支：
  - 通道不变时使用恒等映射
  - 通道变化或降采样时使用 `1x1 Conv`
- 最后与主分支相加，再 `ReLU`

### 7.3 对比公平性说明

建议控制两者的通道规模和深度处于同一量级，不追求参数量完全相同，但不能一边明显更大一边明显更浅，否则结论会失真。

## 8. 训练配置

推荐训练配置如下：

| 项目            | 配置                    |
| ------------- | --------------------- |
| 输入尺寸          | `128 x 128`           |
| Batch Size    | `32`                  |
| Epoch         | `30`                  |
| Optimizer     | `Adam`                |
| Learning Rate | `1e-3`                |
| Weight Decay  | `1e-4`                |
| Loss          | 加权 `CrossEntropyLoss` |
| Scheduler     | `ReduceLROnPlateau`   |
| Early Stop    | 不建议用于主对比；可只保存最佳验证权重   |
| Random Seed   | `42`                  |
| Device        | `cuda` 优先，否则 `cpu`    |

补充建议：

- 每个 epoch 记录一次完整的 train/val 指标
- 统一保存 `best_val_acc` 和 `last_epoch` 两套权重
- 若训练时间允许，可重复 `3` 次实验取均值，以减少随机性影响

## 9. 评估指标与可视化

### 9.1 必做指标

每个 epoch 至少记录：

- `Train Loss`
- `Val Loss`
- `Train Accuracy`
- `Val Accuracy`

### 9.2 建议补充指标

- 混淆矩阵
- Precision
- Recall
- F1-score

### 9.3 可视化输出

至少绘制以下图表：

1. `Plain CNN` 与 `Res CNN` 的训练损失曲线对比
2. `Plain CNN` 与 `Res CNN` 的验证损失曲线对比
3. `Plain CNN` 与 `Res CNN` 的训练准确率曲线对比
4. `Plain CNN` 与 `Res CNN` 的验证准确率曲线对比

建议将同类指标画在同一张图中，便于直接比较。

## 10. 实验输出物

最终建议输出以下内容：

### 10.1 模型结果

- Plain CNN 最佳验证结果
- Res CNN 最佳验证结果
- 两者在测试集上的最终结果

### 10.2 图表结果

- Loss 对比图
- Accuracy 对比图
- 混淆矩阵图

### 10.3 实验结论

围绕以下问题给出结论：

- 残差结构是否让训练更稳定
- 残差结构是否让收敛更快
- 残差结构是否提升验证集表现
- 当网络加深后，普通 CNN 是否更容易出现优化困难

## 12. 执行步骤建议

建议按下面顺序推进：

1. 编写数据加载与分层切分代码
2. 实现 `Plain CNN`
3. 实现 `Res CNN`
4. 固定训练配置，分别训练两种模型
5. 记录并保存每个 epoch 的指标
6. 绘制对比曲线
7. 使用 `test_set` 做最终测试
8. 完成实验结论分析

## 13. 最终推荐结论

如果目标是完成一份结构清晰、对比有效、结论可信的课程/实验项目，最合适的方案是：

- 采用 `128 x 128` 输入
- 从 `training_set` 中分层划分 `train/val`
- 使用加权交叉熵解决类别不平衡
- 设计同量级深度的 `Plain CNN` 与 `Res CNN`
- 固定训练超参数，只改变网络结构
- 以 `Loss/Accuracy` 曲线和最终测试表现作为核心证据

这套方案既能严格对应 `readme` 中的任务要求，也能保证后续实现和实验分析有明确落点。
